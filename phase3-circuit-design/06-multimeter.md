## 1. 멀티미터란?

### 정의

멀티미터(Multimeter)는 전압(Voltage), 전류(Current), 저항(Resistance) 등 여러 전기적 특성을 **하나의 장비**로 측정할 수 있는 만능 측정기입니다. "멀티(Multi, 여러 가지)" + "미터(Meter, 측정기)"에서 이름이 유래했습니다.

전자 회로 디버깅에서 가장 기본적이고 가장 자주 사용하는 도구입니다.

### 소프트웨어 비유

> 멀티미터는 소프트웨어 개발에서의 **`print` 디버깅 / `console.log`** 와 같습니다.
>
> 코드가 안 될 때 `console.log`로 변수 값을 찍어보듯이,
> 회로가 안 될 때 멀티미터로 전압, 저항, 연결 상태를 **눈으로 볼 수 있게** 해줍니다.
>
> ```
> 소프트웨어 디버깅:
>   console.log(variable)     → 변수 값을 화면에 출력
>   debugger / breakpoint     → 실행을 멈추고 상태 확인
>
> 하드웨어 디버깅:
>   멀티미터로 전압 측정      → 회로의 전압을 숫자로 확인
>   멀티미터로 통전 테스트    → 연결 상태를 소리로 확인
>
> 둘 다 "보이지 않는 것을 보이게 만드는" 도구!
> ```
>
> ```python
> # 소프트웨어 디버깅
> def debug_software():
>     print(f"전원 변수 = {power_supply}")    # 값이 맞나?
>     print(f"연결 상태 = {is_connected}")    # 연결됐나?
>     print(f"저항값 = {resistance}")          # 설정이 맞나?
>
> # 하드웨어 디버깅 (멀티미터로 하는 것)
> def debug_hardware():
>     measure_voltage(VCC, GND)        # 전원이 맞나? → 3.3V?
>     continuity_test(wire_A, wire_B)  # 연결됐나? → 삐 소리?
>     measure_resistance(R1)           # 저항값이 맞나? → 10kΩ?
> ```

## 2. 멀티미터의 주요 기능

### 멀티미터 외형과 다이얼

```
  ┌─────────────────────────────────┐
  │        [ LCD 디스플레이 ]         │
  │          1.234                   │
  │                                  │
  │            다이얼                 │
  │         ╱───────╲                │
  │       ╱  OFF      ╲              │
  │     ╱ Ω    ◁│    V~ ╲           │
  │    │ )))  ─┼─  V⎓   │          │
  │     ╲  mA        A  ╱           │
  │       ╲            ╱             │
  │         ╲────────╱               │
  │                                  │
  │  ┌────┐  ┌────┐  ┌────┐        │
  │  │ COM│  │ VΩ │  │ A  │        │
  │  │(흑)│  │(적)│  │(적)│        │
  │  └──┬─┘  └──┬─┘  └──┬─┘        │
  └─────┼───────┼───────┼──────────┘
        │       │       │
      검정    빨강    빨강
      프로브  프로브  프로브
      (공통)  (전압/  (전류
              저항)   전용)

  다이얼 기호 설명:
  V⎓  = DC 전압 (직류, 우리가 주로 사용)
  V~  = AC 전압 (교류)
  Ω   = 저항
  ))) = 통전(Continuity) 테스트
  ◁│  = 다이오드 테스트
  A   = 전류 (암페어)
  mA  = 밀리암페어 (작은 전류)
```

### 프로브 연결 기본 규칙

```
  ┌───────────────────────────────────────────────┐
  │            프로브 연결 규칙                      │
  ├───────────────────────────────────────────────┤
  │                                                │
  │  측정 종류        COM(흑)    VΩ(적)    A(적)   │
  │  ─────────────   ────────   ───────   ──────  │
  │  전압 측정        O          O                  │
  │  저항 측정        O          O                  │
  │  통전 테스트      O          O                  │
  │  다이오드 테스트  O          O                  │
  │  전류 측정        O                    O        │
  │                                                │
  │  COM = Common (공통, 항상 연결)                  │
  │  VΩ  = 전압/저항 측정 시 빨간 프로브              │
  │  A   = 전류 측정 시에만 빨간 프로브               │
  │                                                │
  │  주의: 전류 측정 시에는 반드시 A 단자 사용!       │
  │        VΩ 단자에 꽂고 전류를 흘리면 퓨즈 나감!   │
  └───────────────────────────────────────────────┘
```

## 3. DC 전압 측정

### 원리: 병렬 연결

전압 측정은 측정하려는 두 지점에 프로브를 **병렬(parallel)로** 갖다 대는 것입니다. 회로를 끊거나 변경할 필요가 없습니다.

```
  전압 측정 = 병렬 연결 (회로를 끊지 않음)

  ┌──────────────────────────────┐
  │                              │
  │   측정 대상 회로:             │
  │                              │
  │   VCC (3.3V) ────[R]──── GND │
  │       │                  │   │
  │       │    ┌────────┐    │   │
  │       └────┤ V 멀티 ├────┘   │
  │       적   │ 미터   │  흑    │
  │     프로브  └────────┘ 프로브  │
  │                              │
  │   → 회로는 그대로 동작        │
  │   → 멀티미터가 전압만 "엿봄"  │
  └──────────────────────────────┘

  소프트웨어 비유:
  병렬 측정 = 읽기 전용(read-only) 접근
  → 회로(프로그램)를 변경하지 않고 상태만 확인
  → console.log()가 프로그램 동작에 영향을 주지 않는 것과 같음
```

### 측정 방법

**단계 1: 다이얼을 DC 전압(V⎓)으로 설정**

```
  ESP32 작업 시 주로 측정하는 전압 범위:
  - 3.3V (ESP32 로직 레벨)
  - 5V   (USB 전원, 일부 센서)
  - 0V ~ 3.3V (GPIO 출력, 아날로그 신호)

  오토레인지 멀티미터: 다이얼을 V⎓에 놓으면 자동 선택
  수동레인지 멀티미터: 20V 레인지 선택 (3.3V보다 큰 가장 가까운 값)

  레인지 선택 규칙:
  ┌───────────────────────────────────────┐
  │  예상 전압     →   선택할 레인지       │
  │  0 ~ 0.2V     →   200mV 또는 2V      │
  │  0 ~ 2V       →   2V 또는 20V        │
  │  0 ~ 5V       →   20V               │
  │  0 ~ 12V      →   20V               │
  │  모르겠다      →   가장 높은 레인지    │
  └───────────────────────────────────────┘

  팁: 레인지를 모르면 가장 높은 레인지부터 시작해서
      점점 낮춰가면 안전합니다.
```

**단계 2: 프로브 연결**

```
  ESP32에서 3.3V 전원 확인하기:

  ESP32 보드
  ┌────────────────────┐
  │                    │
  │  3V3 ●────────────┼──── 빨간 프로브 (+)
  │                    │
  │  GND ●────────────┼──── 검정 프로브 (-)
  │                    │
  └────────────────────┘

  디스플레이에 표시되는 값:
  ┌──────────────┐
  │    3.28 V    │   ← 정상 (3.2V ~ 3.4V 범위)
  └──────────────┘

  만약 표시값:
  - 3.2V ~ 3.4V  → 정상
  - 0V           → 전원이 안 들어오고 있음!
  - 5V           → 잘못된 핀에 연결했거나 레귤레이터 문제
  - 2V 이하      → 전원 부족 (USB 케이블/포트 문제 가능)
```

### 실전: GPIO 핀 전압 확인

```
  ESP32 GPIO 핀의 출력을 확인하기:

  코드:
  ┌──────────────────────────────┐
  │  digitalWrite(GPIO2, HIGH); │  → GPIO2에 3.3V 출력
  │  digitalWrite(GPIO2, LOW);  │  → GPIO2에 0V 출력
  └──────────────────────────────┘

  측정:
  ┌────────────────────┐
  │  ESP32             │
  │                    │
  │  GPIO2 ●──────────┼──── 빨간 프로브
  │                    │
  │  GND   ●──────────┼──── 검정 프로브
  │                    │
  └────────────────────┘

  결과:
  - HIGH일 때: 약 3.3V 표시   → 정상
  - LOW일 때:  약 0V 표시     → 정상
  - HIGH인데 0V:  코드 문제 또는 핀 번호 착각
  - HIGH인데 1.5V: 부하가 너무 크거나 쇼트 의심
```

### 전압 측정 시 주의사항

```
  1. 검정 프로브(COM)를 먼저 GND에 연결하는 습관
     → 감전 방지, 측정 기준점 확보

  2. 프로브를 쇼트시키지 않기
     → 두 프로브가 서로 닿으면 안 됨 (전압 측정 시엔 괜찮지만 습관적으로)

  3. 음수 값이 나오면?
     → 빨강/검정 프로브가 반대로 연결된 것
     → 프로브를 바꾸거나, 부호를 무시하고 절대값만 읽으면 됨

  4. "OL" 또는 "1" 표시?
     → Over Limit, 선택한 레인지보다 전압이 높음
     → 더 높은 레인지로 변경
```

## 4. 저항 측정

### 원리: 회로에서 분리 후 측정

저항 측정에서 가장 중요한 규칙은 **반드시 회로에서 분리한 후** 측정하는 것입니다.

```
  왜 분리해야 하는가?

  회로에 연결된 상태:
                    ┌───[R2 1kΩ]───┐
  VCC ──[R1 10kΩ]──┤               ├── GND
                    └───[R3 4.7kΩ]─┘

  R1을 측정하고 싶은데, R2와 R3가 병렬로 연결되어 있으면
  멀티미터는 R1 + (R2 || R3)의 합성 저항을 측정합니다.
  → 실제 R1 값(10kΩ)이 아닌 엉뚱한 값이 나옴!

  더 큰 문제: 회로에 전원이 들어와 있으면
  → 멀티미터의 측정 전류 + 회로의 전류가 섞임
  → 완전히 엉뚱한 값이 나옴
  → 멀티미터가 손상될 수도 있음!
```

### 측정 방법

```
  단계 1: 전원 차단
  단계 2: 측정할 저항을 회로에서 한쪽 다리 이상 분리
  단계 3: 다이얼을 Ω(저항)으로 설정
  단계 4: 두 프로브를 저항의 양쪽 다리에 접촉

  ┌───────────────────────────────────────┐
  │                                       │
  │   분리된 저항:                         │
  │                                       │
  │   빨간 프로브 ──┤ R (10kΩ) ├── 검정 프로브  │
  │                                       │
  │   디스플레이:                           │
  │   ┌──────────────┐                    │
  │   │   9.97 kΩ    │  ← 10kΩ 저항      │
  │   └──────────────┘     (오차 ±5%)     │
  │                                       │
  └───────────────────────────────────────┘

  저항은 극성이 없으므로 프로브 방향은 상관 없습니다.
```

### 저항 컬러코드 확인

```
  저항의 색 띠로 값을 읽고, 멀티미터로 실제 값을 확인:

  예시: 갈-검-주-금 → 10 x 1000 = 10kΩ (오차 ±5%)

  멀티미터 측정값: 9.85kΩ ~ 10.15kΩ → 정상!
  만약 0Ω 이 나오면 → 저항이 쇼트(합선)된 것
  만약 OL 이 나오면 → 저항이 끊어진(개방) 것

  레인지 선택 가이드:
  ┌─────────────────────────────────┐
  │  예상 저항값    →  레인지 선택   │
  │  0 ~ 200Ω      →  200Ω        │
  │  0 ~ 2kΩ       →  2kΩ         │
  │  0 ~ 20kΩ      →  20kΩ        │
  │  0 ~ 200kΩ     →  200kΩ       │
  │  0 ~ 2MΩ       →  2MΩ         │
  │  모르겠다       →  높은 것부터  │
  └─────────────────────────────────┘
```

### 주의: 손가락으로 리드를 잡지 말 것

```
  잘못된 측정:                    올바른 측정:

    (손가락)                      (프로브 끝만 접촉)
    ╱╲
   │  │                          빨강 ──┤ R ├── 검정
   빨강 ──┤ R ├── 검정
    │  │
    (손가락)

  사람의 몸도 저항이 있습니다! (수 kΩ ~ 수백 kΩ)
  손가락으로 저항의 다리와 프로브를 함께 잡으면
  → 몸의 저항이 병렬로 연결됨
  → 실제보다 낮은 값이 나옴
  → 특히 고저항(100kΩ 이상)에서 오차가 큼
```

## 5. 통전 테스트 (Continuity Test)

### 원리: 두 점이 전기적으로 연결되어 있는가?

통전 테스트는 "이 두 점이 연결되어 있는가?"를 확인하는 기능입니다. 연결되어 있으면 **"삐~" 소리**가 나고, 끊어져 있으면 소리가 나지 않습니다.

```
  통전 테스트의 원리:

  멀티미터가 작은 전류를 흘려보냄
  → 전류가 통과하면 (저항이 매우 낮으면) → 삐~ 소리
  → 전류가 안 통하면 (저항이 높으면)     → 소리 없음

  판정 기준: 보통 저항이 약 30Ω ~ 50Ω 이하이면 "연결됨"

  ┌─────────────────────────────────────────────┐
  │  결과          │  의미                        │
  │  ─────────────│──────────────────────────── │
  │  삐~ (소리)    │  연결됨! (전류가 흐름)        │
  │  조용           │  끊어짐 (전류가 안 흐름)      │
  │  삐...삐...     │  접촉 불량 (간헐적 연결)      │
  └─────────────────────────────────────────────┘
```

### 브레드보드 디버깅에 가장 유용!

통전 테스트는 브레드보드에서 회로를 만들 때 **가장 자주 사용하는 기능**입니다.

```
  브레드보드에서 흔한 문제들:

  문제 1: 점퍼선이 제대로 꽂히지 않음
  ┌─────────────────────────────┐
  │  a ● ● ● ● ●               │
  │  b ● ●[점퍼선]● ●          │  ← 겉보기엔 연결됨
  │  c ● ● ● ● ●               │
  │  d ● ● ● ● ●               │
  │  e ● ● ● ● ●               │
  └─────────────────────────────┘
  하지만 실제로는 접촉이 안 되는 경우가 많음!
  → 통전 테스트로 "삐~" 소리가 나는지 확인

  문제 2: 브레드보드 내부 접점 불량
  → 오래된 브레드보드는 특정 행의 접점이 느슨해질 수 있음
  → 통전 테스트로 같은 행의 홀들이 연결되어 있는지 확인

  문제 3: 전원 레일 끊김
  (+) ━━━━━━━━━ ┃ ━━━━━━━━━   ← 일부 브레드보드는
  (-) ━━━━━━━━━ ┃ ━━━━━━━━━      중간에 끊어져 있음!
                 ↑
           여기서 끊어짐!

  → 전원 레일의 양 끝을 통전 테스트로 확인해야 함
```

### 측정 방법

```
  단계 1: 전원 차단 (중요!)
  단계 2: 다이얼을 ))) (통전) 모드로 설정
  단계 3: 두 프로브를 확인할 두 지점에 접촉

  예시: 점퍼선이 제대로 연결되었는지 확인

         프로브 A (빨강)             프로브 B (검정)
            │                           │
            ▼                           ▼
  ┌─────────────────────────────────────────────┐
  │  a ●──●──●──●──●   │   ●──●──●──●──● a    │
  │  b ●──●  [점퍼선]  │   ●──●──●──●──● b    │
  │  c ●──●──●──●──●   │   ●──●──●──●──● c    │
  │  d ●──●──●──●──●   │   ●──●──●──●──● d    │
  │  e ●──●──●──●──●   │   ●──●──●──●──● e    │
  └─────────────────────────────────────────────┘

  "삐~" → 점퍼선이 제대로 연결됨!
  조용   → 점퍼선이 빠져 있거나 접촉 불량!
```

### 활용 시나리오

```
  시나리오 1: LED가 안 켜질 때
  ─────────────────────────
  ESP32 → 저항 → LED → GND 경로에서
  각 구간을 통전 테스트:

  ESP32 GPIO ──[구간1]── 저항 ──[구간2]── LED ──[구간3]── GND

  구간1 확인: GPIO 핀 ~ 저항 한쪽 다리
  구간2 확인: 저항 다른쪽 다리 ~ LED 긴 다리(+)
  구간3 확인: LED 짧은 다리(-) ~ GND 레일

  소리가 안 나는 구간 = 끊어진 곳!


  시나리오 2: I2C 통신이 안 될 때
  ──────────────────────────────
  ESP32 SDA(GPIO21) ~ 센서 SDA 핀: 통전 확인
  ESP32 SCL(GPIO22) ~ 센서 SCL 핀: 통전 확인
  ESP32 3V3 ~ 센서 VCC: 통전 확인
  ESP32 GND ~ 센서 GND: 통전 확인

  하나라도 소리가 안 나면 → 배선 문제!


  시나리오 3: 쇼트(합선) 확인
  ───────────────────────────
  VCC와 GND 사이에 통전 테스트
  → "삐~" 소리가 나면 어딘가 합선됨!
  → 전원을 연결하면 과전류로 부품 손상 위험!
  → 브레드보드 배선을 하나씩 확인
```

## 6. 전류 측정

### 원리: 직렬 연결

전류 측정은 전압 측정과 완전히 다릅니다. 회로를 **끊고** 멀티미터를 **직렬(series)로** 연결해야 합니다.

```
  전류 측정 = 직렬 연결 (회로를 끊어야 함!)

  원래 회로:
  VCC ──────────[R]────────── GND

  전류 측정을 위해 끊고 삽입:
  VCC ───┐                ┌──[R]──── GND
         │  ┌──────────┐  │
         └──┤ A 멀티   ├──┘
         적 │  미터    │ 흑
        프로브 └──────────┘ 프로브

  전류가 멀티미터를 "통과"해서 흘러야 측정 가능!

  소프트웨어 비유:
  직렬 측정 = 미들웨어(middleware) / 프록시(proxy)
  → 데이터(전류)가 반드시 미들웨어를 통과하도록 끼워넣기
  → 통과하면서 양을 측정
```

### !! 매우 중요한 주의사항 !!

```
  ╔══════════════════════════════════════════════════════════════╗
  ║                                                              ║
  ║   절대로 전류 측정 모드에서 전압처럼 병렬 연결하지 마세요!     ║
  ║                                                              ║
  ║   전류 모드의 멀티미터 내부 저항은 거의 0Ω입니다.             ║
  ║   병렬로 연결하면 = VCC와 GND를 직접 쇼트시키는 것!           ║
  ║                                                              ║
  ║                                                              ║
  ║   잘못된 연결 (절대 금지!):                                   ║
  ║                                                              ║
  ║   VCC ─────────[R]─────────── GND                            ║
  ║     │                          │                             ║
  ║     └──┤ A 멀티미터 ├──────────┘   ← 병렬 = 쇼트!           ║
  ║                                                              ║
  ║   결과:                                                      ║
  ║   → 과전류 발생 (수 A의 전류가 순간적으로 흐름)               ║
  ║   → 멀티미터 내부 퓨즈가 끊어짐 (퓨즈 교체 필요)             ║
  ║   → 심하면 회로의 부품이 손상될 수도 있음                     ║
  ║   → 전원 공급 장치의 보호 회로가 동작할 수 있음               ║
  ║                                                              ║
  ╚══════════════════════════════════════════════════════════════╝
```

### 올바른 전류 측정 방법

```
  ESP32 회로의 전체 전류 소비 측정:

  단계 1: 프로브를 COM과 A(또는 mA) 단자에 연결
  단계 2: 다이얼을 전류 모드(A 또는 mA)로 설정
  단계 3: 회로의 전원선을 끊고 멀티미터를 직렬 삽입

  USB ──(+)──┐        ┌──(+)── ESP32 VIN
             │  [A]   │
             └──┤├───┘
                멀티미터
         (전류가 여기를 통과)

  USB ──(-)──────────────(-)── ESP32 GND
         (GND는 그대로 연결)

  디스플레이:
  ┌──────────────┐
  │   78.5 mA    │  ← ESP32 대기 전류 (WiFi OFF)
  └──────────────┘

  참고값:
  - ESP32 Deep Sleep:  약 10uA
  - ESP32 대기:        약 80mA
  - ESP32 WiFi 활성:   약 120~240mA
  - ESP32 WiFi 전송:   약 180~240mA (피크)
```

### 레인지 선택과 퓨즈

```
  멀티미터에는 보통 두 개의 전류 단자가 있음:

  ┌──────────────────────────────────────────┐
  │                                          │
  │  mA 단자: 작은 전류 (0 ~ 200mA 또는     │
  │           0 ~ 400mA)                     │
  │           → 퓨즈: 보통 200~500mA         │
  │           → ESP32 회로에 적합             │
  │                                          │
  │  A 또는 10A 단자: 큰 전류 (0 ~ 10A)     │
  │           → 퓨즈: 보통 10A               │
  │           → 모터, 파워 LED 등에 사용      │
  │                                          │
  │  팁: 전류 크기를 모르면 A(10A) 단자에서   │
  │      먼저 측정하고, 값이 작으면 mA 단자로  │
  │      옮기면 안전합니다.                    │
  │                                          │
  └──────────────────────────────────────────┘
```

## 7. 다이오드 테스트 모드

### 원리

다이오드 테스트 모드는 다이오드(또는 LED)의 **순방향 전압 강하(Forward Voltage Drop)** 를 측정합니다. 다이오드가 정상인지, 방향이 맞는지 확인할 수 있습니다.

```
  다이얼을 ◁│ (다이오드) 모드로 설정

  순방향 측정 (정상):
  빨강(+) ──▶│── 검정(-)
  (애노드)        (캐소드)

  디스플레이: 0.5V ~ 0.7V (실리콘 다이오드)
             1.8V ~ 3.2V (LED, 색상에 따라 다름)


  역방향 측정:
  빨강(+) ──│◀── 검정(-)
  (캐소드)        (애노드)

  디스플레이: OL (Over Limit, 전류가 안 흐름 = 정상)


  고장 판별:
  ┌──────────────────────────────────────────┐
  │  측정 결과          │ 상태                │
  │  ──────────────────│────────────────── │
  │  순방향: 0.5~0.7V  │ 정상 (실리콘)       │
  │  순방향: 1.8~3.2V  │ 정상 (LED)          │
  │  순방향: 0V         │ 쇼트됨 (고장)       │
  │  순방향: OL         │ 개방됨 (고장)       │
  │  양방향 모두 0V     │ 완전히 쇼트 (고장)  │
  │  양방향 모두 OL     │ 완전히 개방 (고장)  │
  └──────────────────────────────────────────┘
```

### LED 테스트

```
  LED도 다이오드이므로 같은 방법으로 테스트 가능!

  보너스: 대부분의 멀티미터에서 LED가 실제로 살짝 켜집니다!
  → LED가 살아있는지 빠르게 확인 가능
  → LED의 극성(+/-)을 모를 때 방향도 확인 가능

  빨강 프로브를 긴 다리(+)에, 검정 프로브를 짧은 다리(-)에
  → LED가 켜지고 전압이 표시되면 정상!
  → 아무 반응 없으면 프로브를 반대로 → 그래도 안 되면 고장

  LED 색상별 순방향 전압:
  ┌────────────────────────────┐
  │  색상      │  순방향 전압   │
  │  ─────────│────────────  │
  │  빨강      │  1.8 ~ 2.2V  │
  │  주황      │  2.0 ~ 2.2V  │
  │  노랑      │  2.0 ~ 2.4V  │
  │  초록      │  2.0 ~ 3.0V  │
  │  파랑      │  3.0 ~ 3.5V  │
  │  백색      │  3.0 ~ 3.5V  │
  └────────────────────────────┘
```

## 8. 디버깅 시나리오: 회로가 안 될 때 어디를 측정할까?

### 체계적 디버깅 체크리스트

회로가 동작하지 않을 때, 아래 순서대로 체크하면 대부분의 문제를 찾을 수 있습니다.

```
  ╔══════════════════════════════════════════════════════════════╗
  ║           회로 디버깅 체크리스트 (순서대로!)                   ║
  ╠══════════════════════════════════════════════════════════════╣
  ║                                                              ║
  ║  □ 체크 1: 전원이 제대로 공급되는가?                          ║
  ║  □ 체크 2: 신호가 나오고 있는가?                              ║
  ║  □ 체크 3: 연결이 끊어진 곳은 없는가?                         ║
  ║  □ 체크 4: 저항값이 맞는가?                                   ║
  ║  □ 체크 5: 부품이 고장나지 않았는가?                          ║
  ║                                                              ║
  ╚══════════════════════════════════════════════════════════════╝

  소프트웨어 비유:
  이것은 "로그 기반 디버깅 프로세스"와 같습니다.
  버그를 찾을 때도 체계적으로 접근하듯이,
  회로도 순서대로 확인하면 반드시 원인을 찾을 수 있습니다.

  if (프로그램 안 됨) {
      check("환경변수 설정됐나?");       // → 전원 확인
      check("함수가 호출되고 있나?");    // → 신호 확인
      check("import/require 빠진 건?"); // → 연결 확인
      check("설정값이 맞나?");          // → 저항값 확인
  }
```

### 체크 1: 전원이 제대로 공급되는가?

```
  가장 먼저 확인! 전원이 없으면 아무것도 안 됩니다.

  측정 방법: DC 전압 모드, VCC-GND 간 전압 측정

  측정 지점들:
  ┌────────────────────────────────────────────────┐
  │                                                │
  │  ① USB 전원 출력 확인                          │
  │     USB 5V 핀 ~ GND: 4.8V ~ 5.2V 예상         │
  │                                                │
  │  ② ESP32 보드 전원 확인                         │
  │     ESP32 3V3 핀 ~ GND: 3.2V ~ 3.4V 예상      │
  │                                                │
  │  ③ 브레드보드 전원 레일 확인                     │
  │     (+) 레일 ~ (-) 레일: 3.2V ~ 3.4V 예상     │
  │     주의: 레일의 양쪽 끝 모두 확인!              │
  │                                                │
  │  ④ 센서/모듈의 VCC 핀 확인                      │
  │     센서 VCC ~ 센서 GND: 기대 전압 확인         │
  │                                                │
  └────────────────────────────────────────────────┘

  ASCII 측정 다이어그램:

       USB          ESP32           센서
      ┌───┐      ┌────────┐      ┌──────┐
  5V──┤   ├──VIN─┤        ├─3V3──┤ VCC  │
      │   │      │        │      │      │
  GND─┤   ├──GND─┤        ├─GND──┤ GND  │
      └───┘      └────────┘      └──────┘

  측정① ──┤V├──        측정③ ──┤V├──
   5V~GND             센서VCC~GND
        측정② ──┤V├──
         3V3~GND

  기대값 vs 실제값:
  ┌─────────────────────────────────────────────┐
  │  측정 지점       │  기대값    │  비정상        │
  │  ───────────────│──────────│──────────── │
  │  USB 출력        │  ~5V     │  0V → 케이블   │
  │  ESP32 3V3      │  ~3.3V   │  0V → 보드?    │
  │  전원 레일       │  ~3.3V   │  0V → 점퍼선?  │
  │  센서 VCC       │  ~3.3V   │  0V → 배선?    │
  └─────────────────────────────────────────────┘
```

### 체크 2: 신호가 나오고 있는가?

```
  전원이 OK라면, 다음은 신호를 확인합니다.

  GPIO 핀 전압 측정:
  ┌────────────────────────────────────────────────────┐
  │                                                    │
  │  ESP32 코드에서 GPIO2를 HIGH로 설정했다면:          │
  │  → GPIO2 핀 ~ GND 간 전압 = ~3.3V 예상            │
  │  → 0V가 나오면? 코드 문제 또는 핀 번호 오류!        │
  │                                                    │
  │  PWM 출력 (analogWrite):                           │
  │  → DC 전압으로 대략적 확인 가능                     │
  │  → 50% 듀티 → 약 1.65V 표시 (평균값)              │
  │  → 정확한 PWM 파형은 오실로스코프 필요              │
  │                                                    │
  │  I2C/SPI 통신 핀:                                  │
  │  → 풀업 저항이 있으면 대기 시 ~3.3V                 │
  │  → 0V이면 풀업 저항 누락 또는 배선 문제             │
  │                                                    │
  └────────────────────────────────────────────────────┘

  ESP32            LED 회로
  ┌────────┐
  │        │      ┌───[R 330Ω]───[LED]───┐
  │  GPIO2 ├──────┘                       │
  │        │                              │
  │  GND   ├──────────────────────────────┘
  └────────┘
     ↑                ↑          ↑
    측정A            측정B      측정C
   (GPIO출력)     (저항 뒤)   (LED 뒤)

  측정A: GPIO2~GND = 3.3V? → 신호 출력 확인
  측정B: 저항 뒤~GND = 약 1.3V? → 전압 강하 확인
  측정C: LED(-)~GND = 약 0V? → GND 연결 확인
```

### 체크 3: 연결이 끊어진 곳은 없는가?

```
  통전 테스트로 경로 확인! (전원 OFF 상태에서)

  전체 경로를 구간별로 나누어 통전 테스트:

  ESP32 GPIO ──[A]── 점퍼선 ──[B]── 저항 ──[C]── LED ──[D]── GND

  구간A: ESP32 핀 ~ 점퍼선 한쪽  → 삐? (핀이 제대로 꽂혔나)
  구간B: 점퍼선 반대쪽 ~ 저항    → 삐? (점퍼선이 연결됐나)
  구간C: 저항 ~ LED              → 삐? (같은 행에 있나)
  구간D: LED ~ GND 레일          → 삐? (GND가 연결됐나)

  끊어진 구간을 찾으면 → 그 부분의 배선을 수정!

  자주 끊어지는 곳:
  1. 점퍼선이 홀에 제대로 꽂히지 않은 경우
  2. 브레드보드의 다른 행에 꽂은 경우 (흔한 실수!)
  3. 브레드보드 중앙 홈을 걸쳐서 꽂은 경우
  4. 전원 레일이 중간에 끊어진 브레드보드
```

### 체크 4: 저항값이 맞는가?

```
  잘못된 저항값은 회로 오동작의 흔한 원인입니다.

  확인 방법: 전원 OFF → 저항 분리 → Ω 모드로 측정

  흔한 실수:
  ┌─────────────────────────────────────────────┐
  │                                             │
  │  1. 컬러코드를 잘못 읽음                     │
  │     갈-검-빨-금 = 1kΩ인데                    │
  │     갈-검-주-금 = 10kΩ으로 잘못 사용         │
  │     → 10배 차이!                             │
  │                                             │
  │  2. 저항 부품함에서 잘못 꺼냄                 │
  │     330Ω이 필요한데 3.3kΩ을 사용             │
  │     → LED가 매우 어둡게 켜짐                 │
  │                                             │
  │  3. 풀업/풀다운 저항값 오류                   │
  │     I2C 풀업에 10MΩ을 사용 (너무 큼)         │
  │     → 통신 실패                              │
  │     적절한 값: 4.7kΩ ~ 10kΩ                  │
  │                                             │
  └─────────────────────────────────────────────┘
```

### 체크 5: 부품이 고장나지 않았는가?

```
  다이오드/LED 테스트:
  → 다이오드 모드로 순방향 전압 확인
  → 0V 또는 OL이면 고장

  의심되는 부품은 회로에서 분리 후 개별 테스트!

  트랜지스터 간이 테스트:
  → 다이오드 모드로 B-E, B-C 간 전압 확인
  → NPN: B→E, B→C 모두 ~0.6V, 역방향 OL → 정상
```

## 9. 안전 주의사항

### 기본 안전 규칙

```
  ╔══════════════════════════════════════════════════════════════╗
  ║              멀티미터 사용 안전 수칙                          ║
  ╠══════════════════════════════════════════════════════════════╣
  ║                                                              ║
  ║  1. 측정 전에 모드와 레인지를 확인하라                        ║
  ║     → 전류 모드인 줄 모르고 전압을 재면 퓨즈 나감!           ║
  ║                                                              ║
  ║  2. 전류 측정 시 절대 병렬 연결하지 마라                      ║
  ║     → 반드시 직렬 연결! (가장 흔한 실수)                     ║
  ║                                                              ║
  ║  3. 저항/통전 측정 시 반드시 전원을 끄라                      ║
  ║     → 전원이 켜진 상태에서 저항을 재면 값이 틀림              ║
  ║     → 멀티미터가 손상될 수 있음                               ║
  ║                                                              ║
  ║  4. 높은 전압(>60V)은 초보자가 측정하지 마라                  ║
  ║     → ESP32 프로젝트는 보통 3.3V~12V이므로 안전              ║
  ║     → 절대 콘센트(220V AC) 측정하지 말 것!                   ║
  ║                                                              ║
  ║  5. 프로브 상태를 정기적으로 확인하라                          ║
  ║     → 절연 피복이 벗겨진 프로브는 교체                        ║
  ║     → 프로브 끝이 산화되면 접촉 불량 발생                     ║
  ║                                                              ║
  ║  6. 측정 후에는 다이얼을 OFF로 돌려라                         ║
  ║     → 배터리 소모 방지                                       ║
  ║     → 다음 사용 시 잘못된 모드에서 시작하는 것 방지           ║
  ║                                                              ║
  ╚══════════════════════════════════════════════════════════════╝
```

### ESP32 프로젝트에서의 안전

```
  ESP32 프로젝트에서 멀티미터를 사용할 때 특별히 주의할 점:

  1. ESP32의 GPIO는 3.3V 로직!
     → 5V를 GPIO에 연결하면 ESP32 손상
     → 전압 분배기 회로를 측정해서 3.3V 이하인지 확인

  2. ESP32의 GPIO 전류 제한
     → GPIO 한 핀에서 최대 약 40mA (권장 20mA)
     → 전류 측정으로 과전류 여부 확인 가능

  3. USB 전원 측정
     → USB 포트의 전류 제한: 보통 500mA
     → ESP32 + 센서 + 액추에이터의 총 전류가
        500mA를 넘지 않는지 확인

  4. 리튬 배터리 프로젝트
     → 배터리 극성 확인 시 전압 모드 사용
     → 절대 쇼트시키지 않기!
     → 배터리 충전 전압 모니터링: 4.2V (만충전)
```
